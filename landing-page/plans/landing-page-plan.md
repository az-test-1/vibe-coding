# GitHub Landing Page 계획서

## 프로젝트 개요
YouTube 러닝 세컨드 브레인 - 유튜브 영상을 자동으로 구조화하고 검색 가능하게 만드는 개인용 지식 자동화 서비스

---

## 1. 목표 및 목적

### 1.1 목표
- 프로젝트의 핵심 가치 제안을 명확하게 전달
- 타겟 사용자(지식 노동자)의 공감대 형성
- MVP 기능과 차별점 강조
- 베타 테스트 참여 유도

### 1.2 타겟 오디언스
- 유튜브로 학습하는 지식 노동자 (재문, 지혜, 세훈 같은 페르소나)
- 영상 기반 학습 효율을 높이고 싶은 사용자
- "본 영상의 정확한 구간"을 다시 찾지 못해 고민하는 사용자

---

## 2. 페이지 구조 및 섹션

### 2.1 Hero Section (상단 메인 영역)
**목적**: 즉각적인 가치 제안과 문제 인식

**구성 요소**:
- **메인 헤드라인**: 
  - "유튜브 영상, 이제 검색 가능하게"
  - "본 영상의 그 부분, 5초 안에 찾기"
- **서브 헤드라인**: 
  - "AI가 자동으로 요약하고, 하이라이트하고, 검색 가능하게 만듭니다"
- **CTA 버튼**: 
  - "베타 테스트 참여하기" (메인)
  - "더 알아보기" (보조)
- **비주얼**: 
  - 서비스 UI 스크린샷 또는 GIF (영상 플레이어 + 하이라이트 리스트)
  - 또는 간단한 인터랙티브 데모

**디자인 톤**:
- 깔끔하고 모던한 느낌
- 학습/지식 관련 이미지나 일러스트
- 강조 색상: 신뢰감 있는 블루 또는 그린 계열

---

### 2.2 Problem Section (문제 정의)
**목적**: 사용자의 Pain Point를 명확히 제시

**구성 요소**:
- **섹션 제목**: "이런 경험 있으신가요?"
- **문제 리스트** (아이콘 + 텍스트):
  1. "어디서 본 내용인지 기억이 안 나요"
  2. "타임스탬프를 수동으로 기록하기 너무 번거로워요"
  3. "메모/북마크를 해도 결국 다시 찾는 데 실패해요"
  4. "영상은 텍스트처럼 검색이 안 되죠"
- **통계/인용**: 
  - "학습→정리→복습 사이클이 평균 70% 더 오래 걸립니다" (가능하면)

**디자인**:
- 다크 배경 또는 대비가 강한 섹션
- 각 문제를 카드 형태로 시각화

---

### 2.3 Solution Section (해결책)
**목적**: 우리가 어떻게 문제를 해결하는지 설명

**구성 요소**:
- **섹션 제목**: "AI가 자동으로 처리합니다"
- **핵심 기능 4-5개** (카드 레이아웃):
  1. **자동 요약**
     - 아이콘: 📝
     - 설명: "OpenAI가 영상 전체를 분석해 핵심 내용을 요약합니다"
  2. **자동 하이라이트**
     - 아이콘: ⏱️
     - 설명: "의미 단위로 구간을 나누고 타임스탬프를 자동 생성합니다"
  3. **검색 가능**
     - 아이콘: 🔍
     - 설명: "키워드, 문장, 질문으로 원하는 구간을 즉시 찾습니다"
  4. **개인 지식 저장소**
     - 아이콘: 💾
     - 설명: "본 날짜, 핵심 문장, 메모가 자동으로 저장됩니다"
  5. **즉시 재생**
     - 아이콘: ▶️
     - 설명: "검색 결과를 클릭하면 해당 구간으로 바로 이동합니다"

**디자인**:
- 그리드 레이아웃 (2열 또는 3열)
- 각 카드에 아이콘 + 제목 + 설명
- 호버 효과로 인터랙션 추가

---

### 2.4 How It Works Section (작동 방식)
**목적**: 사용자 여정을 단계별로 설명

**구성 요소**:
- **섹션 제목**: "3단계로 시작하세요"
- **단계별 플로우** (타임라인 스타일):
  1. **YouTube URL 입력**
     - 설명: "복사한 영상 링크를 붙여넣으세요"
     - 이미지: URL 입력 화면
  2. **AI 자동 분석**
     - 설명: "AI가 transcript를 분석해 요약, 하이라이트, 키워드를 생성합니다"
     - 이미지: 처리 중 화면 또는 프로세스 다이어그램
  3. **검색하고 재학습**
     - 설명: "원하는 내용을 검색하고 해당 구간을 바로 재생하세요"
     - 이미지: 검색 결과 + 영상 플레이어 화면

**디자인**:
- 수평 또는 수직 타임라인
- 각 단계를 번호와 함께 시각화
- 화살표로 플로우 연결

---

### 2.5 Features Section (상세 기능)
**목적**: MVP 기능을 더 자세히 소개

**구성 요소**:
- **섹션 제목**: "주요 기능"
- **기능 카테고리** (탭 또는 아코디언):
  
  **P0 (MVP 필수 기능)**:
  - YouTube URL 입력 → 자동 요약/하이라이트 생성
  - 의미 단위 타임스탬프 자동 생성
  - 텍스트·구간 기반 검색 기능
  - 본 날짜/구간별 메모 자동 저장
  - 키워드 기반 재검색 → 해당 구간 즉시 재생

  **P1 (경험 강화 기능)** - "곧 출시 예정":
  - Notion/Obsidian 자동 페이지 생성 Export
  - 개인별 반복 시청 구간 자동 추출
  - "나중에 다시 보기" AI 추천

**디자인**:
- 탭 또는 아코디언 UI
- 각 기능에 스크린샷 또는 GIF 첨부

---

### 2.6 Benefits Section (기대 효과)
**목적**: 사용자가 얻을 수 있는 구체적 이점

**구성 요소**:
- **섹션 제목**: "이렇게 달라집니다"
- **Before/After 비교**:
  - **Before (기존 방식)**:
    - 수동 타임스탬프 기록
    - 메모 앱에 별도 정리
    - 다시 찾기 어려움
    - 루틴 붕괴로 포기
  - **After (우리 서비스)**:
    - 자동 하이라이트 생성
    - 자동 저장 및 구조화
    - 5초 안에 원하는 구간 찾기
    - 지속 가능한 학습 루틴

- **정량적 지표** (가능하면):
  - 학습→정리→복습 사이클 70% 단축
  - 재시청 검색 성공률 80% 이상
  - 수동 정리 루틴 완전 제거

**디자인**:
- 좌우 또는 상하 비교 레이아웃
- Before는 회색/약한 색상, After는 강조 색상

## 3. 디자인 가이드라인

### 3.1 색상 팔레트
- **Primary**: 신뢰감 있는 블루 (#3B82F6 또는 유사)
- **Secondary**: 그린 또는 퍼플 (강조용)
- **Background**: 화이트 또는 매우 밝은 그레이
- **Text**: 다크 그레이 (#1F2937)
- **Accent**: Primary 색상의 변형

### 3.2 타이포그래피
- **헤드라인**: Bold, 큰 사이즈 (2.5rem 이상)
- **서브헤드라인**: Medium, 중간 사이즈 (1.5rem)
- **본문**: Regular, 읽기 쉬운 사이즈 (1rem)
- **폰트**: 시스템 폰트 또는 Google Fonts (Inter, Noto Sans KR 등)

### 3.3 레이아웃
- **최대 너비**: 1200px (중앙 정렬)
- **섹션 간격**: 충분한 여백 (80px~120px)
- **반응형**: 모바일, 태블릿, 데스크톱 대응
- **그리드 시스템**: 12열 그리드 또는 Flexbox

### 3.4 인터랙션
- **호버 효과**: 버튼, 카드에 부드러운 전환
- **스크롤 애니메이션**: 섹션 진입 시 fade-in (옵션)
- **로딩 상태**: 부드러운 스피너 또는 스켈레톤 UI
